# ✅ Ders Özeti (Lesson Summary)

Tebrikler! Bu modülü tamamladınız. Kursun bu noktasında şunları biliyorsunuz:

* Flask, minimum bağımlılıklarla gelen bir  *microframework* ’tür.
* Uygulamalar geliştirmek için Flask; debugging server’lar, routing, template’ler ve error handling gibi özelliklere sahiptir.
* Flask, bir Python paketi olarak kurulabilir ve topluluk (community) eklentileri kullanılarak genişletilebilir.
* Django, Flask’e kıyasla özellik açısından ağır (feature heavy) bir full stack framework’tür.
* Uygulama yapılandırmasını (application configuration) environment variable’lardan, Python dosyalarından ve doğrudan `app.config` objesinden ayarlayabilirsiniz.
* Flask, her istemci çağrısı için bir *Request* ve bir *Response* objesi sağlar.
* Flask  *Request* ’inden header’lar gibi ek bilgileri alabilirsiniz.
* *Request* objesini parse ederek query parametrelerini ve body’yi alabilirsiniz.
* Yanıtı (response) istemciye göndermeden önce *Response* objeleri üzerinde status ayarlayabilirsiniz.
* RESTful endpoint’ler oluşturmak için *dynamic routes* kullanabilirsiniz.
* HTTP response, isteğin işlendiğinde ne olduğunu belirtmek için bir status code gerektirir.
* Başarıyı, kullanıcı hatasını veya sunucu hatasını gösteren birden fazla HTTP status code sınıfı vardır.
