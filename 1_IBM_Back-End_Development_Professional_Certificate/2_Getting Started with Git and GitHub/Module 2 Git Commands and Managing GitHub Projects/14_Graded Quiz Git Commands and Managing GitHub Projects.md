## Question 1

You need to run three commands to push local changes to the GitHub repository. Which command will you use when your changes are ready to be moved to the GitHub repository?

- [ ] `git remove -v`
- [x] `git push`
- [ ] `git commit -m`
- [ ] `git add`

**Answer:** `git push`

**Explanation:** Correct! The third command needed to push local changes to the GitHub repository is `git push`.

**Score:** 1 / 1 point

## Question 2

What are three commonly used commands for working with remote repositories?

- [ ] create, delete, copy
- [x] push, pull, fetch
- [ ] open, close, send
- [ ] open, edit, push

**Answer:** push, pull, fetch

**Explanation:** Correct! Three commonly used operations for working with remote repositories are push, pull, and fetch.

**Score:** 1 / 1 point

## Question 3

When managing forks, which command can you use to fetch and merge the remote branch in a single step?

- [ ] `git pull downstream`
- [ ] `git fetch upstream`
- [ ] `git merge upstream/main`
- [x] `git pull <upstream>`

**Answer:** `git pull <upstream>`

**Explanation:** Correct! When managing forks, you can use the `git pull <upstream>` command to fetch and merge the remote branch in a single step.

**Score:** 1 / 1 point

## Question 4

Which of the following commands is used to make a branch active?

- [ ] `git branch`
- [ ] `got commit`
- [x] `git checkout`
- [ ] `git add`

**Answer:** `git checkout`

**Explanation:** Correct! The `git checkout` command is used to switch the currently active branch. It can also be used to restore files.

**Score:** 1 / 1 point

## Question 5

Which command can a developer use to keep up-to-date with the upstream?

- [ ] `git request pull`
- [x] `git pull <upstream>` and `git fetch <upstream>`
- [ ] `git revert`
- [ ] `git push`

**Answer:** `git pull <upstream>` and `git fetch <upstream>`

**Explanation:** Correct! A developer uses `git pull` and `git fetch` to update their local repository from an upstream remote.

**Score:** 1 / 1 point

## Question 6

In Git workflow, which step involves creating a separate workspace for making changes and developing features without interfering with the main codebase?

- [ ] Initializing a Git repository in your project's local directory
- [x] Creating a branch from the main repository and working on the branch
- [ ] Performing an initial commit after selecting the required files
- [ ] Selecting and moving files to the staging area

**Answer:** Creating a branch from the main repository and working on the branch

**Explanation:** Correct! You create a branch from the main repository and work on the branch. A branch in Git is like a separate workspace where you can make changes and develop features without interfering with the main codebase.

**Score:** 1 / 1 point

## Question 7

Which of the following commands helps you to view the commit history?

- [ ] `git history`
- [ ] `git commit`
- [ ] `git checkout`
- [x] `git log`

**Answer:** `git log`

**Explanation:** Correct! The `git log` command helps you to browse previous changes to a project by enabling you to view the commit history.
