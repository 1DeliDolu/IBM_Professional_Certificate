## Question 1

What is meant by an initial commit in a Git workflow?

- [x] Officially recording the current state of your project’s files in the local Git repository
- [ ] Creating a local copy of the project’s code from the remote repository
- [ ] Retrieving changes made by the team and updating your local repository
- [ ] A request to your team to review and approve changes

**Answer:** Officially recording the current state of your project’s files in the local Git repository

**Explanation:** Correct! An initial commit is the process of officially recording and saving the current state of your project’s files in the local Git repository. This step marks the starting point of your project’s version history.

**Score:** 1 / 1 point

## Question 2

Which git command allows you to see the state of your working directory and the staged snapshot of your changes?

- [ ] `git reset`
- [ ] `git log`
- [ ] `git checkout`
- [x] `git status`

**Answer:** `git status`

**Explanation:** Correct! The `git status` command allows you to see the state of your working directory and the staged snapshot of your changes.

**Score:** 1 / 1 point

## Question 3

Why do you fork a project?

- [ ] To add a branch to an existing tree in the project
- [ ] To give a new user permission to the project
- [ ] To create a backup of the main branch of the project
- [x] To use an existing repository as the base for a new project

**Answer:** To use an existing repository as the base for a new project

**Explanation:** Correct! Forking allows you to modify or extend a repository without affecting the original repository. Frequently, developers use forking to make an existing repository the starting point of your new project.

**Score:** 1 / 1 point

## Question 4

Which of the following options should be used to fork a repository?

- [ ] `git copy`
- [ ] `git fork`
- [ ] Click the “Copy” button on the project page
- [x] Navigate to the GitHub project and select **Fork** at the top of the page.

**Answer:** Navigate to the GitHub project and select **Fork** at the top of the page.

**Explanation:** Correct! There are no Git commands used for forking a project. You can fork it by navigating to the GitHub project and selecting the Fork option.

**Score:** 1 / 1 point
